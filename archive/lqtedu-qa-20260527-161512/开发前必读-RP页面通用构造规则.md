# QA 文档快速索引

## AI_FAST_READ

read_policy:
- 不要一上来通读所有文档，先按任务类型跳转。
- 每份文档前面的 `AI_FAST_READ` 是快速决策入口；后文是细节仓库。
- 真要改代码时，文档只能定方向，最终仍以源码、接口、截图、RP 为准。

task_router:
| 任务 | 先看 | 目的 |
| --- | --- | --- |
| 新建或修改前端页面/模块 | `lqtedu-dev-workflow` Skill；本 QA 只保留开发判断索引 | 完整角色编排、subagent 创建、授权暂停、Developer 分工、Reviewer 和 Reporter 输出以 Skill 为准；使用 Skill 只授权 subagent 做只读探索、交叉核对和方案编排，不等于授权修改代码 |
| 写 RP 页面 / 改 UI | `开发前必读-RP页面开发规则.md` | 判壳层、技术栈、参考实现、交互边界 |
| 找组件/颜色/表格/控件写法 | `base-ebase组件认知地图.md` | 快速定位 `base.jsp` / `ebase.jsp` 示例和 CSS 来源 |
| 改完要验收 / 写测试结论 | `测试前必读-RP页面测试规则.md` | 按入口、参数、显隐、交互、视觉输出结论 |
| 碰数字中枢 | `数字中枢结构速记.md` | 查管理页、静态大屏、接口链路和坑点 |
| 班主任手册 mock 转接口 | `班主任手册-假数据接口对接清单.md` | 对齐假数据命名空间、占位接口、字段语义 |

default_sequence:
1. 先读本索引。
2. 再读对应文档的 `AI_FAST_READ`。
3. 搜源码验证关键事实。
4. 按 `AI 受控前端开发流程` 判断证据等级、停机条件、影响面和安全边界。
5. 能改才改代码；不能改就输出已确认内容、缺失证据、风险和待确认问题。
6. 回看对应文档的 checklist。

maintenance_rules:
- 新增“怎么判断、怎么选实现、怎么落结构”：写进开发规则。
- 新增“开发任务怎么安排角色、调用后先创建哪些 agent、怎么授权后再改、怎么分工复查、真实 subagent 何时必须使用”：优先写进 `lqtedu-dev-workflow` Skill；QA 只保留索引和开发知识，不重复维护完整人员安排。
- 新增“证据等级、停机条件、影响面、安全边界、字段来源、Mock 边界、交付审计”：写进开发规则。
- 新增“怎么验证、怎么写结论、异常数据、并发边缘、结论门禁”：写进测试规则。
- 新增 `base/ebase` 组件、样式、截图证据：写进组件认知地图。
- 新增数字中枢入口、目录、接口、链路、坑点：写进数字中枢速记。
- 新增班主任手册接口字段、mock 映射、对接顺序：写进接口对接清单。
