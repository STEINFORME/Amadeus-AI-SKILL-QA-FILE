# 数字中枢结构速记

## AI_FAST_READ

scope:
- 数字中枢不是单个前端页面，是 `JSP 管理页 -> /digitalHub 静态大屏 -> 后端接口`。
- 不要裸开 `web/digitalHub/*.html` 下结论；必须从管理页启动并带加密 `school_id`。

entry_chain:
1. 管理页在 `web/department/office/campus/DataHub/*.jsp`。
2. 点击“启动页面”。
3. JSP 调 `/digitalHub/get_en_school_id` 拿加密 `school_id`。
4. JSP `window.open('/digitalHub/*.html?school_id=...&currentTime=0&musicSwitch=true&rgbswitch=true&')`。
5. 静态页读 URL 参数，再调 `/digitalHub/*` 或 `/zhpj/report/*`。

quick_map:
| 管理页 | 静态页 | 优先查 |
| --- | --- | --- |
| `DataHub-index.jsp` | `/digitalHub/index.html` | `getMain`、`getPicList`、`getBaseInfo`、`getFrequentNumber`、`getTsInfo` |
| `DataHub-xszhpj.jsp` | `/digitalHub/xszhpj/xszhpj.html` | `get_digital_hub_sets`、`get_digital_hub_data`、`get_zh_pic_stu_rec` |
| `DataHub-bjzp.jsp` | `/digitalHub/bjzhpb.html` | `getDySetting`、`getReportListByRateId`、`getReportDetail` |
| `DataHub-bjzp-daily.jsp` | `/digitalHub/bjzhpb-daily.html` | `getDyDailySetting`、`getDyDailyReport` |
| `DataHub-index-Pic.jsp` | 无直接大屏 | `getPicList`、`savePicList` |

debug_order:
1. 管理页入口能不能进。
2. `school_id` 是否由入口页生成且已加密。
3. 静态页 URL 参数是否完整。
4. 接口是否返回。
5. 最后才看图表、轮播、头像、视觉。

known_traps:
- `DataHub`、`digitalHub`、`DigtalHub`、`get_zh_digital_hut_data` 同时存在，别按正常命名猜。
- 学生综评核心数据很多在 `report_service` / `ZhpjReportUtils`，不是全在 `DigitalHubService`。
- `web/digitalHub/` 里有测试页、demo 素材、旧结构目录，不要顺手乱改。
- 首页 `COURSE_TYPE -> 校本课程显示` 当前链路未完全打透，动首页时必须重新顺源码。

## 1. 先记结论

- 这套东西不是单纯的前端页面，而是三段式：`JSP 管理页 -> /digitalHub 静态大屏 -> /digitalHub 后端接口`。
- 真正的大屏入口不在 `web/digitalHub/*.html` 里直接点开，而是在 `web/department/office/campus/DataHub/*.jsp` 里点击“启动页面”。
- 静态页必须带加密后的 `school_id` 才能正常工作。裸开 html，看到空白、缺块、接口报错，都不算有效结论。
- 命名非常乱：`DataHub`、`digitalHub`、`DigtalHub`、`get_zh_digital_hut_data` 都同时存在。别对这套命名抱幻想，按实际路径查。

## 2. 目录分层

| 层级 | 位置 | 作用 |
| --- | --- | --- |
| 管理页 | `web/department/office/campus/DataHub/` | 后台配置页、启动页、图片配置页 |
| 静态大屏 | `web/digitalHub/` | 真正展示的大屏 html、css、js、图片、音乐 |
| 学生综评子模块 | `web/digitalHub/xszhpj/` | 学生综评专页和 4 个局部组件 |
| 后端控制器 | `src/com/ajax/openAPI/DigitalHubController.java` | `/digitalHub/*` 路由入口 |
| 后端服务 | `src/com/ajax/openAPI/DigitalHubService.java` | 基础数据、配置、图片、列表等查询/保存 |
| 综评数据能力 | `ZhpjReportUtils` | `get_digital_hub_*`、`get_zh_pic_stu_rec` 等综评核心数据 |

## 3. 入口链路

1. 管理页点击“启动页面”。
2. JSP 先调用 `/digitalHub/get_en_school_id`。
3. 后端把当前学校 `school_id` 加密后返回。
4. JSP 再 `window.open('/digitalHub/*.html?school_id=...&currentTime=0&musicSwitch=true&rgbswitch=true&')`。
5. 静态页从 URL 参数里读取 `school_id`。
6. 静态页直接请求 `/digitalHub/*` 或 `/zhpj/report/*` 接口取数。

## 12. 易踩坑

- 裸开 `web/digitalHub/*.html` 基本没有意义，必须带加密 `school_id`。
- `school_id` 不是明文，入口页先调 `/digitalHub/get_en_school_id`。
- 不是所有数字中枢接口都走 `DigitalHubService`，学生综评那块很多逻辑在 `ZhpjReportUtils`。
- 页面命名和函数命名有明显手误：
  - `goToDigtalHub`
  - `get_zh_digital_hut_data`
- 当前仓库里 `web/digitalHub/` 有测试页、demo 素材、旧结构目录，不要上来就把不相关页面一起动了。
- 查问题时优先按“管理页 -> 静态页 -> controller -> service/report_service”这个顺序走，别一头扎进静态页里乱翻。

## 13. 逻辑顺序速记

### 13.1 通用顺序

1. 先在 `DataHub-*.jsp` 管理页看当前模块能改什么。
2. 点击“启动页面”时，JSP 先调 `/digitalHub/get_en_school_id` 取加密 `school_id`。
3. JSP 再 `window.open('/digitalHub/*.html?...')` 打开对应静态页。
4. 静态页从 URL 读取 `school_id/currentTime/musicSwitch/rgbswitch`。
5. 静态页按自己的模块逻辑去调 `/digitalHub/*` 或 `/zhpj/report/*` 接口。
6. `DigitalHubController` 负责接参数、解密 `school_id`、转到 `DigitalHubService` 或 `report_service/ZhpjReportUtils`。
7. 前端把接口结果映射到具体卡片、榜单、轮播区、图标区。

### 13.2 首页 `DataHub-index.jsp -> index.html`

1. 管理页先通过 `getMain` 读主配置，通过 `getPicList` 读轮播图。
2. 管理页允许改两类东西：
   - `COURSE_TYPE`：学校概况里的“校本课程统计范围”
   - 图片列表：首页轮播/展示图
3. 点击启动后进入 `web/digitalHub/index.html`。
4. 首页静态页会继续拉：
   - `getSchoolUserCount`
   - `get_zh_digital_hut_data`
   - `getFrequentNumber`
   - `getTsInfo`
   - `getBaseInfo`
   - `getPicList`
   - `get_digital_hub_dict`
5. 页面再把这些数据分别灌到学校概况、高频行为、导师制/综评相关块、轮播图片区。
6. 首页里的子页面跳转会继续带着 `school_id/currentTime/musicSwitch/rgbswitch` 往下传。

### 13.3 学生综评 `DataHub-xszhpj.jsp -> xszhpj.html`

1. 管理页先调 `/zhpj/report/get_digital_hub_sets` 拉左右两栏当前配置。
2. 修改“上榜范围”时，走 `/zhpj/report/save_digital_hub_eval_sets`，保存：
   - `eval_id`
   - `eval_score_type`
   - `eval_grade`
3. 修改“排行榜 / 上榜人数 / 平均时长”时，走 `/zhpj/report/save_digital_hub_sets`，保存：
   - `RANK_TYPE`
   - `RANK_PERSON_COUNT`
   - `RANK_SECONDS`
4. 启动后进入 `web/digitalHub/xszhpj/xszhpj.html`。
5. 静态页通过 `Promise.all` 取：
   - `get_digital_hub_sets`
   - `get_digital_hub_data`
   - `get_digital_hub_dict`
6. 页面把 `set0/hub_data0` 映射成左榜，把 `set1/hub_data1` 映射成右榜。
7. 左榜当前选中学生会再调 `get_zh_pic_stu_rec`，更新中间学生画像。

### 13.4 班级综评 `DataHub-bjzp.jsp -> bjzhpb.html`

1. 管理页先调 `getDySetting` 读取当前配置，再按 `RATE_ID` 取报表列表。
2. 管理页能改的核心项：
   - `RATE_ID`：评价体系
   - `TYPE/REPORT_ID`：取最新报表还是自主指定报表
   - `NAME1`：文明班级榜标题
   - `SOURCE + detailList`：文明班级榜取数规则
   - `NAME2`：班级得分榜标题
3. 启动后进入 `web/digitalHub/bjzhpb.html`。
4. 静态页顺序是：
   - `getDySetting`
   - `getReportListByRateId`
   - `getReportDetail`
5. 页面根据 `SOURCE + detailList` 先生成左侧滚动名单，再把报表明细渲染成主榜表格。

### 13.5 班级综评日榜 `DataHub-bjzp-daily.jsp -> bjzhpb-daily.html`

1. 管理页先调 `getDyDailySetting` 读取整套日榜配置。
2. 管理页能改的核心项：
   - `RATE_ID`：评价体系
   - `NAME`：榜单名称
   - `SOURCE`：前一天 / 当天数据
   - `TYPE`：显示评分 / 显示图标
   - `GOOD_MIN_INCLUDE,GOOD_MIN,NORMAL_MIN_INCLUDE,NORMAL_MIN`：图标分档规则
   - `GOOD_ICON,NORMAL_ICON,BAD_ICON`：三档图标
   - `SIZE`：图标尺寸
3. 启动后进入 `web/digitalHub/bjzhpb-daily.html`。
4. 静态页通过 `getDailyData()` 取“当天报表 + 当前设置”。
5. 页面把 `NAME` 拼到顶部标题，把 `TYPE` 决定成“显示数字”还是“显示图标”。
6. 图标模式下，再用档位阈值和三张图标去决定每个单元格显示哪张图。
7. `SOURCE=0` 时按代码逻辑每天 6 点更新；`SOURCE=1` 时每 10 分钟更新，并带 A/B 缓冲预取切换。

## 15. 这份速记现在能不能支撑“重构前极快速理解”

可以，但边界要说清楚，别把文档神化了。

- 现在这份文档已经足够支撑我在重构前 `5~10 分钟建立全貌`：
  - 模块入口在哪
  - 管理页和展示页怎么对应
  - 关键配置保存到哪里
  - 静态页从哪些接口取数
  - 出问题时第一批该翻哪些文件
- 它已经能让我避免最蠢的误判：
  - 不会去裸开 `html` 得出假结论
  - 不会把 `DigitalHubService` 当成所有逻辑的唯一来源
  - 不会把“管理页能改什么”和“展示页显示什么”混成一锅
- 但它还不是“完全替代源码”的东西。真正开始重构某一屏时，还是要继续读这一屏对应的：
  - 管理页 JSP
  - 静态页 html/js
  - controller
  - service 或 `ZhpjReportUtils`
- 当前唯一明确标红的未完全打通点是首页 `COURSE_TYPE -> 校本课程显示` 这条链。这个地方文档已经写明“不确定”，后续真动首页时必须再顺一遍，不能装懂。

换句话说，这份文档现在已经够做“快速接管地图”，不够做“闭眼盲改许可证”。前者已经达标，后者本来也不该存在，这项目还没配有那么奢侈。
