### 对接时优先做的 mapper

common mapper:
- `normalizeTerm(row)`：统一 `CODE/TERM_ID`、`NAME/TERM_NAME`。
- `normalizeClass(row)`：统一 `CLASS_ID`、`CLASS_NAME`、`GRADE_ID`、`TEACHER_ID`、`FTEACHER_ID`。
- `normalizeDynamicField(detail)`：统一模板字段和填写字段：`id`、`name`、`textType`、`required`、`minLength`、`maxLength`、`commentEnable`、`comment`、`options`、`content`。
- `buildDetailArrayStr(fields)`：把页面动态字段值回写成后端要求的 `detailArrayStr` 字符串。
- `buildPicArrayStr(files)` / `buildPicDelArrayStr(files)`：图片新增和删除字符串。
- `normalizeDispatchBatch(row)`：统一周/月批次和详情批次显示。
- `normalizeBooleanStatus(res)`：统一 `status/message`，Vue3 页面失败走 `ElNotification`。

term/class 默认值规则:
- 班主任个人端 `headmaster/guide.jsp` 和 `headmaster/dispatch-worksheet.jsp` 都是两个选择框：先班级、后学期；切班级时必须先重算该班级可用学期，再请求列表。
- 工作导览每个分组的“添加”入口只由后端 `showAdd/SHOW_ADD` 控制，前端不要再叠加“是否当前学期”的判断；如果不能新增，后端应直接返回 `showAdd=0`。
- 工作导览记录行的“删除”入口只由后端记录字段 `IS_OWN/is_own` 控制，前端不要再叠加“是否当前学期”的判断；如果不能删除，后端应直接返回 `IS_OWN=0` 或后续提供明确删除权限字段。
- `headmaster/add-worksheet.jsp` 是复用页：从“工作导览-添加”进入时没有 `actId/indexId`，属于自建新增，`wbDoType=1`，可以显示自建工作单类型标签；从“派发工作单”列表进入时带 `indexId`，属于派发填写，`wbDoType=2`，类型已由派发记录定死，不能显示可切换类型标签。
- 班主任个人端页面跳转参数只传页面定位必需值：新增自建单只传 `classId/termCode/className/termName/from`；工作导览已有记录查看/编辑才传 `actId`；个人派发工作单进入填写页只传 `indexId/classId/termCode/className/termName/from`，不要把派发列表行 `ID` 当作个人记录 `actId` 传入。`className/termName/from` 只用于页面展示和原地返回，不进接口。`wbDoType` 由是否存在 `indexId` 推导，`wbType/type` 不要从入口 URL 传给添加/查看页，添加/查看页自身也不要读取 URL 的 `wbType/type` 做类型兜底。
- 班主任个人端派发工作单列表 `/getTeacherSendList` 状态以 `SUCCESS_STATUS` 为准：`-1=未完成`、`0=待完成`、`1=已完成`；已完成行打开 `view-worksheet.jsp`，未完成/待完成行打开 `add-worksheet.jsp`。旧字段 `OVER_TIME` 只能作为缺少 `SUCCESS_STATUS` 时的兜底，不再作为主状态来源。
- 添加工作单页的快捷语库是字段级能力：每个字段各自判断，只有编辑态且字段 `COMMENT_ENABLE=1` 时显示按钮；按钮在字段标题同行靠右，hover 展开已有快捷语，点击某条后把该条内容追加到当前字段输入框末尾，不覆盖已有内容，也不要自动加换行。按钮显隐只看 `COMMENT_ENABLE`，不要因为快捷语列表为空或前端未匹配到内容就隐藏。
- 添加工作单页字段类型按后端 `TEXT_TYPE` 决定控件：`S_LINE_SHORT` 用 450px 短 `el-input`；`S_LINE` 用铺满一行的 `el-input`；`M_LINE` 用铺满一行的 `el-input type=textarea`；`SELECT` 用项目标准 `el-select` 120px；`DATE_TEXT` 用项目标准日期框 180px。文本类字段必须绑定 `maxlength` 并开启 `show-word-limit`，让字数上限显示在输入框右侧。
- 添加工作单页字段 `IS_REQUIRED=1` 时，字段标题文字右侧显示红色 `*`，颜色用 `var(--mainRed)`，与标题上下居中。
- 添加工作单页提交校验必须用 `el-form` / `el-form-item` 字段级校验反馈；必填项提示显示在对应控件下方，不用 `ElNotification` 代替字段校验。
- 添加工作单页提交成功后，在当前窗口原地跳回打开它的来源页；`from=guide` 回工作导览，`from=dispatch` 回派发工作单，不刷新 opener、不关闭当前窗口、不重新开新窗口。
- 部门检查工作单页字段编辑态参考个人端查看工作单的可编辑态：按 `TEXT_TYPE` 渲染短单行、长单行、多行、下拉、日期控件，必填和字数限制走 `el-form` / `el-form-item` 字段级校验；无编辑权限时保持原文字查看样式，不显示输入框。
- 部门检查工作单页切换已完成/未完成人员时，如果当前内容已修改：开关开启则先自动保存再切换；开关关闭则弹 200px `el-popconfirm`，点“是”保存后切换，点“否”丢弃修改后切换。取消关闭页面时如有未提交修改，也用同类 `el-popconfirm` 判断是否保存。
- 部门检查工作单页提交成功提示固定为 `操作成功`，不是 `保存成功`；提交成功后关闭当前页并刷新来源页面。
- `/getWbCheck` 真实返回与原始文档有偏差：自动保存开关在 `data.autoSaveMap.POWER_OPEN`，开关 id 在 `data.autoSaveMap.ID`；已完成记录的字段内容在 `successList[].updateList.details[]`，不是原文档示例里的 `successList[].details[]`；快捷语库在 `successList[].updateList.quick_comment_list[]`；记录可编辑权限看 `successList[].can_edit`。
- 部门模板管理的“快捷语库”参考 `department/dyc/tspdf/setting_vue3.jsp`：列表页按钮跳转到独立快捷语库页面，不在模板管理页内做大弹框；快捷语库页面再按原版方式展示项目列表，并在“设置”中编辑单个项目语库。
- 添加页类型标签只用于自建新增场景，数据只能来自后端返回的自建工作单类型及其字段配置；如果后端没有返回某类型字段配置，前端不要为了凑标签自己塞空表单或给 `/getTeacherOwnSheetInfo` 追加 `wbType` 参数。
- 同时拿到 `nowTerm` 和 `termList` 时，`termCode` 只能从当前班级或当前页面的 `termList` 中取；`nowTerm` 只是优先候选，命中 `termList` 才能选中。
- 如果 `nowTerm` 不在当前 `termList` 内，默认选 `termList[0]`，不要把不存在的 `nowTerm` 人工塞进下拉框。
- 这个兜底必须发生在列表查询前；首次接口返回后才拿到 `classList/termList` 的页面，要先校正 `classId/termCode`，如果和本次请求参数不同，立刻用校正后的参数重查一次，不渲染无效学期数据。

`/getTeacherOwnSheetInfo` 返回结构:
- 请求参数固定只有 `indexId`、`actId`、`termCode`、`classId`、`wbDoType`，不要传 `wbType`。
- `wbDoType=2` 或 `actId` 不为空时，返回常规填写/编辑结构：`data.act_type`、`data.act_type_name`、`data.pic_list[]`、`data.updateList.details[]`、`data.updateList.quick_comment_list[]`、`data.classId`、`data.can_edit`。
- `wbDoType=1` 且 `actId` 为空时，返回自建新增结构：`data.teacherSheetlist[]`。每项是一种自建工作单类型，包含 `WB_TYPE`、`WB_TYPE_NAME`、`classId`、`LIMIT_MIN`、`LIMIT_MAX`、`picLimit`、`picLimitMin`、`details[]`、`quick_comment_list[]`。
- 自建新增页顶部类型标签必须来自 `teacherSheetlist[]`，初始状态不默认选中任何类型；用户点击标签后，才切换该项的 `details[]` 和当前 `WB_TYPE`，不额外请求 `/getTeacherOwnSheetInfo`，也不要靠 URL `wbType` 当接口参数。

### 明确风险点

- `/getTeacherOwnSheetInfo` 文档没有 `wbType` 参数，添加/查看初始化请求都不要额外传；`wbType` 只出现在 `/updateTeacherOwnSheet`，但文档说明写成“学期code”，语义仍需后端确认。
- `/getWbCheck` 参数表里 `wbId/indexId` 说明有括号截断，按实际调用校准。
- `/updateDeptSendDetail` 参数说明有截断：`wbUserType/endTime/cancelWb` 都是可选修改项，空值代表不改，`cancelWb=0` 代表取消后续派发。
- `/swapDetail` 的 `preId/currentId` 语义容易反，必须用真实接口点一次上移/下移确认。
- `/addTemp`、`/updateTemp` 的 `selectList` 只在 `SELECT` 类型传，值是 JSON 字符串，不是数组对象。
- 图片字段当前只看到字符串契约，真实上传链路要另查项目已有上传组件，不要自己造上传协议。
- 部门“某班工作概览”的 `classData` 只有 `ACT_ID/INDEX_ID/WB_DO_TYPE`，没有派发主体 `wbId`；如果继续跳 `check.jsp`，派发记录的检查参数可能不完整。真实“查看”应优先按 `/getDeptTeacherRecordInfo(actId)` 落地，但必须先确认 RP 是弹层还是独立页。

## 对接风险和待确认

1. 源文档所有请求参数都写成“否”，但保存类接口显然有必填项。前端表单必须按业务校验，不要被文档带歪。
2. `wbUserType/WB_USER_TYPE` 是派发单“填写权限”，不是菜单权限。菜单权限仍然按部门端模板配置、部门端派发管理两类能力判断。
3. `wbTimeObj`、`wbSendObj`、`selectList` 都是 JSON 字符串。提交前统一 `JSON.stringify`，别传对象。
4. 时间格式源数据同时出现 `2026/06/30` 和 `2025-09-01 00:00:00`，页面展示和比较前要集中 normalize。
5. `getDeptSendDetail` 的 `START_DATE/END_DATE` 中文说明在源文档里疑似写反，实际以后端样例字段名为准。
6. 旧版只覆盖部门端模板和部门端派发管理；2026-04-29 已补齐班主任个人端、部门工作汇总、自建工作单、检查页接口索引，实际入口以上方全量目录为准。
