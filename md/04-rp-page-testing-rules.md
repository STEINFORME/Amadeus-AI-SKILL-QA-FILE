# RP 页面测试规则摘要

## 适用范围

用于 LQTedu 页面修改后的验证、结论书写、视觉异常排查和 E2E 落点判断。

开发前判断看 `02-rp-page-development-rules.md`。

## 默认验证顺序

1. 入口：页面能否从正确入口进入。
2. 参数：URL、上下游参数、关键 id 是否对齐。
3. 显隐逻辑：权限、状态、hover、空态、只读/编辑态。
4. 交互状态：点击、hover、focus、active、popconfirm、提交前校验。
5. 视觉细节：间距、颜色、字号、边框、阴影。

入口没通、参数不对时，不要继续写视觉结论。

## 结论标签

- `通过`
- `有问题`
- `受限完成`
- `非本次范围`
- `未验证`

存在关键未验证项时，不允许写 `通过`。

## 阻断口径

以下情况停在入口或参数层：

- 登录进不去。
- 页面 500。
- 缺关键参数。
- 依赖接口没通。
- 真实保存、删除、上传、提交无法在当前环境安全执行。

此时结论写清楚 `未验证：需提交`、`未验证：需造数` 或具体阻断原因。

## 受控验证新增项

安全检查：

- `v-html` / `innerHTML` / JSP 变量直接输出。
- URL 参数拼接、直接跳转、下载地址拼接。
- 只隐藏按钮但仍能触发接口。
- token、手机号、身份证、账号是否进入 URL、console、localStorage/sessionStorage。
- 上传下载限制、失败处理和越权风险。

异常数据：

- `null`、`undefined`、空数组、空字符串、超长文本、特殊字符、HTML 字符串。
- 数字字段返回字符串、字符串字段返回数字、日期为空或格式异常。
- 接口错误码、超时、返回登录页 HTML、无权限、部分成功。
- `total` 与 `list` 不一致、分页越界、图片失败、文件链接失效。

并发和交互边缘：

- 重复点击提交。
- 保存中关闭弹窗。
- 请求未完成时切换筛选条件。
- 后发请求先回时，旧数据是否覆盖新数据。
- 删除当前页最后一条后分页是否合理。
- 编辑后取消是否恢复原数据。
- 弹窗反复打开是否残留旧数据或校验状态。
- 保存成功是否刷新列表，保存失败是否保留输入。

## 视觉异常排查

- 黑线、蓝边、阴影、发灰、发虚：先查 `border`、`outline`、`box-shadow`、`:focus`、`:active`、父容器、伪元素。
- 用户说“附近异常”：组件本体、父容器、相邻元素、分割线都要看。
- hover 需求必须测真实 hover。
- hover 里有 `popconfirm` 时，确认框打开后 hover 区不能塌。
- 不要把开发批注当页面文案。
- 不要没先判断同构就拿参考页硬比。

## 数字中枢测试专项

- 不能裸开 `web/digitalHub/*.html` 下结论。
- 必须从 `web/department/office/campus/DataHub/*.jsp` 的“启动页面”进入。
- 顺序：入口链路 -> 加密 `school_id` -> URL 参数 -> 接口返回 -> 图表/轮播/头像/视觉。

## E2E 落点

- 新业务测试放 `D:\software\e2e\tests\<业务域>\*.spec.ts`。
- 页面动作放 `D:\software\e2e\support\<业务域>\*.page.ts`。
- 开发阶段默认只跑本地或 `gz` 测试云。
- `www`、`w2`、`w3`、`w4` 正式云必须用户明确确认后才能碰。

## 交付前自查

- 是否看过真实 diff。
- 是否验证入口、参数、显隐、交互、视觉。
- 是否覆盖本次字段相关的异常数据。
- 是否检查权限不足和重复提交。
- 是否说明没验证的真实提交、删除、上传或生产环境操作。
- 是否把验证限制和剩余风险写清楚。
