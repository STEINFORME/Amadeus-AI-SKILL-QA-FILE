# 前端 CS 自修改规则摘要

## 适用范围

用于 LQTedu 前端页面中的确定性视觉问题：间距、按钮排列、颜色、圆角、日期展示格式、Base 样式复用、mock 适配层和 Vue helper 冲突。

## 核心规则

- `Base` 页是基础样式参照源。
- RP 提到“新样式”时，优先理解为 `Base` / `EBase` 已有 DOM 或组件样式，不是新增一套 CSS。
- 用户要求“完全复制 Base 页样式”时，先整页或整块复制母版 DOM、class、组件写法，再做业务参数适配。
- 接口未到但页面要先可运行时，做局部数据适配层和 mock 开关，保留真实接口方法名、入参和返回结构。
- Vue3 + Element Plus 的 JSP 页面里，组件脚本如果先于页面主脚本加载，不要在组件文件顶层重复声明 `ref`、`computed`、`watch`、`onMounted` 等 Vue helper。

## 间距

| 场景 | 间距 |
| --- | --- |
| 标题与内容 | `25px` |
| 内容与底部按钮 | `25px` |
| 表格上方元素与表格 | `8px` |
| 各类按钮之间 | `5px` |
| 同一个按钮内图标与文字 | `3px` |

同组元素优先让父容器用 `display: flex/grid` + `gap` 控制；块级结构距离、RP 固定距离、旧样式兼容或特殊偏移再用 `margin`。

## 颜色和圆角

- 颜色按 `Base` 页。
- 能使用项目已有 token、class、变量时，不手写新的十六进制颜色。
- 所有方框、弹框都要有圆角。
- 弹框及以上的大块容器圆角：`5px`。
- 大方按钮及以下的小控件圆角：`2px`。

## 日期展示

- 页面展示日期统一把 `-` 改成 `/`。
- 示例：`2026-04-29` 展示为 `2026/04/29`。
- 这里只管展示格式；接口入参或后端存储要求 `-` 时不能擅自改。

## Base 整块复用

- 如果目标页是已有页面的变体，先复制源页面和组件文件，再做最小业务适配。
- “完全复制样式”包括 DOM 结构、组件类型、class、按钮 type、内联布局和提示样式。
- 不要自行重排、换控件、换按钮 class 或补一套局部 CSS。

## Mock 适配层

- 真实接口未到场时，用 `USE_MOCK` 这类显式开关。
- 用 `xxxApi` 适配对象集中承接下拉、上传、生成模板、确认导入、历史记录等调用。
- mock 数据要贴近后续接口结构：字段名、数组层级、状态字段、历史记录结构先按页面消费形态固定。
- 后端接口到场后，只替换适配对象内部实现。
- 不要让页面默认打必失败的真实接口。

## 不能自作主张

以下情况先停：

- 改动会影响接口字段、入参、返回值解析。
- 改动会影响按钮是否显示、是否可点、点击后走什么流程。
- 改动会影响数据筛选、排序、分页、权限、状态判断。
- 改动需要改公共组件、公共样式或多个页面联动，但没有明确依据。
- `Base` 页、RP、截图、现网页面之间互相冲突。

## 自检清单

- 是否对照 `Base` 页同类控件。
- 标题与内容是否 `25px`。
- 内容与底部按钮是否 `25px`。
- 表格上方元素与表格是否 `8px`。
- 按钮之间是否 `5px`。
- 按钮内图标与文字是否 `3px`。
- 同组元素是否优先用父容器 `gap`。
- 颜色是否复用项目已有规范。
- 方框、弹框、按钮圆角是否符合规则。
- 日期展示是否统一为 `/`。
- 本次修改是否停留在样式和展示层，没有偷偷改业务逻辑。
